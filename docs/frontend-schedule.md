# BI 图表平台前端项目计划

## 1. 计划说明

本计划面向 BI 图表平台前端从零建设，范围覆盖模板体系、设计态、运行态、折线图、柱状图、散点图、表格、图层和统计指标能力。计划重点说明各阶段具体完成内容。

## 2. 总体排期

| 阶段 | 时间 | 阶段目标 |
| --- | --- | --- |
| 第 1 阶段 | 第 1 周 | 完成前端工程初始化和页面骨架 |
| 第 2 阶段 | 第 2 周 | 完成模板体系和统一配置模型 |
| 第 3 阶段 | 第 3 周 | 完成折线图、柱状图和混合图的基础展示 |
| 第 4 阶段 | 第 4 周 | 完成散点图和表格展示能力 |
| 第 5 阶段 | 第 5 周 | 完成图层能力和统计指标能力 |
| 第 6 阶段 | 第 6 周 | 完成联调、收口和交付准备 |

## 3. 分阶段计划

### 3.1 第 1 周：工程与页面骨架

| 目标类别 | 具体内容 |
| --- | --- |
| 工程初始化 | 搭建前端工程、基础依赖、开发环境、构建流程 |
| 页面框架 | 搭建设计态页面和运行态页面骨架 |
| 布局结构 | 建立看板区域、图表卡片区域、配置面板区域的基本布局 |
| 公共能力 | 建立基础路由、样式体系、接口封装和公共目录结构 |

### 3.2 第 2 周：模板体系与配置模型

| 目标类别 | 具体内容 |
| --- | --- |
| 模板体系 | 定义折线图、柱状图、散点图、表格、混合图模板 |
| 模板能力 | 明确每类模板支持的维度数量、指标数量、展示方式和默认配置 |
| 配置模型 | 建立数据配置、维度配置、指标配置、图层配置、统计指标配置、布局配置统一结构 |
| 配置界面骨架 | 建立右侧配置面板分区，包括基础配置区、图层配置区、维度配置区、指标配置区、统计配置区 |

### 3.3 第 3 周：折线图、柱状图、混合图

| 目标类别 | 具体内容 |
| --- | --- |
| 笛卡尔图渲染 | 完成折线图、柱状图和混合图的统一渲染结构 |
| 折线图能力 | 支持多指标、左右轴、折线/面积切换、图例、提示框 |
| 柱状图能力 | 支持并列柱状、堆叠柱状、分类对比 |
| 混合图能力 | 支持同一图中同时配置折线、面积和柱状序列 |
| 预览能力 | 配置变更后可以实时看到图表结果 |

### 3.4 第 4 周：散点图与表格

| 目标类别 | 具体内容 |
| --- | --- |
| 散点图能力 | 完成 X/Y 指标映射、分类分组、点位展示、提示框和图例 |
| 表格能力 | 完成动态列头、维度与指标列映射、分页和表格视图 |
| 视图切换 | 支持图表视图与表格视图切换 |
| 模板接入 | 将散点图和表格纳入统一模板体系和统一配置入口 |

### 3.5 第 5 周：图层与统计指标

| 目标类别 | 具体内容 |
| --- | --- |
| 图层能力 | 支持图层新增、编辑、启停，以及维度、指标、统计指标绑定图层 |
| 图层切换 | 在运行态支持图层切换，并驱动图表重新计算展示结果 |
| 统计指标能力 | 支持均值线、1σ 区间、2σ 区间、分位点线 |
| 标准差窗口 | 支持用户输入滚动 N 年窗口，用于标准差区间计算 |
| 分位点能力 | 支持当前值在滚动窗口中的百分位位置展示，而不是固定分位数 |

### 3.6 第 6 周：联调与交付收口

| 目标类别 | 具体内容 |
| --- | --- |
| 全流程联调 | 打通模板创建、配置编辑、预览展示、运行态展示完整流程 |
| 配置整理 | 检查模板默认配置、配置项一致性和图层关联关系 |
| 展示收口 | 调整图表样式、表格展示效果和页面布局细节 |
| 交互收口 | 检查提示框、图例、缩放、图层切换等交互表现 |
| 构建交付 | 完成前端构建验证和交付资料整理 |

## 4. 里程碑输出

| 里程碑 | 时间点 | 输出内容 |
| --- | --- | --- |
| 里程碑 1 | 第 1 周结束 | 前端工程可运行，设计态和运行态骨架完成 |
| 里程碑 2 | 第 2 周结束 | 模板体系和统一配置模型完成 |
| 里程碑 3 | 第 3 周结束 | 折线图、柱状图、混合图可配置可展示 |
| 里程碑 4 | 第 4 周结束 | 散点图和表格能力完成接入 |
| 里程碑 5 | 第 5 周结束 | 图层和统计指标能力完成 |
| 里程碑 6 | 第 6 周结束 | 前端整体能力联调完成并具备交付条件 |

## 5. 功能建设顺序

| 顺序 | 建设内容 | 说明 |
| --- | --- | --- |
| 1 | 先搭框架 | 先完成工程、页面和配置骨架，保证后续能力有统一承载结构 |
| 2 | 再定模板和配置模型 | 先固定模板和配置结构，再做具体图表能力，避免反复返工 |
| 3 | 先做笛卡尔图 | 折线图、柱状图、混合图共用最多基础能力，应优先建设 |
| 4 | 再做散点图和表格 | 作为差异化展示形态单独补齐 |
| 5 | 最后做图层和统计 | 这部分依赖前面的图表结构和配置结构稳定后再接入更合适 |
| 6 | 收口联调后交付 | 最后一周不再新增核心能力，重点完成联调和表现优化 |
