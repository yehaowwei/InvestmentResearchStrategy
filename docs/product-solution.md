# BI 图表平台产品方案

## ◆ 1. 项目定位

本项目拟建设一套面向 BI 场景的图表平台，目标不是交付若干固定图表页面，而是形成一套可持续扩展的图表生产与展示产品能力。

当前业务在图表建设中普遍存在以下问题：

- 图表类型逐步增多，但配置方式不统一
- 每新增一种图表，都需要重复开发和重复维护
- 业务分析人员难以通过统一界面自主完成图表配置
- 图表展示结果与图表配置过程脱节，复用性较差
- 图层、统计指标等复杂分析能力缺乏统一承载方式

因此，本项目的产品定位是建设一个可配置、可扩展、可复用的图表平台，使图表能够通过模板化方式生产，通过统一界面配置，通过运行态稳定展示。

## ◆ 2. 产品目标

### ● 总体目标

建设一套面向分析场景的图表平台，支持用户完成图表创建、配置、预览和展示，逐步形成统一的图表产品体系。

### ● 本期目标

本期重点不是覆盖全部 BI 平台能力，而是先完成图表平台最核心的产品基础。

| 目标 | 说明 |
| --- | --- |
| 模板体系 | 用统一模板管理不同图表类型 |
| 设计态 | 支持图表创建和配置 |
| 运行态 | 支持图表展示和基础交互 |
| 图表类型 | 支持折线图、柱状图、散点图、表格、混合图 |
| 图层能力 | 支持图层创建、绑定、启停和切换 |
| 统计能力 | 支持均值、标准差区间、分位点等分析能力 |

## ◆ 3. 用户与场景

### ● 目标用户

本项目主要面向三类使用对象：

#### ① 业务分析人员

关注如何快速生成分析图表，减少对开发的依赖，提升日常分析效率。

#### ② 数据运营人员

关注趋势、波动、区间和异常点的观察，希望能够在同一个平台中完成多种图表展示。

#### ③ 管理人员

关注最终展示结果，希望在运行态直接查看图表和分析结果，而不参与复杂配置过程。

### ● 核心使用场景

| 场景 | 说明 |
| --- | --- |
| 图表创建 | 从模板中选择合适图表类型开始配置 |
| 图表配置 | 配置维度、指标、图层、统计项、样式和交互 |
| 图表预览 | 在设计态实时查看配置结果 |
| 图表展示 | 在运行态查看最终图表和表格 |
| 图层切换 | 查看不同图层下的分析结果 |
| 统计观察 | 查看均值、标准差区间、分位点等辅助分析结果 |

## ◆ 4. 产品结构

从产品结构上看，本平台由设计态和运行态两部分组成。

### ● 设计态

设计态负责图表生产，是用户完成图表定义和配置的核心入口。

设计态需要承载以下能力：

- 模板选择
- 数据配置
- 图层配置
- 统计配置
- 样式配置
- 交互配置
- 画布预览
- 布局调整

设计态的目标不是只让用户“改几个字段”，而是让用户通过可视化界面完成一个图表从创建到预览的完整过程。

### ● 运行态

运行态负责图表消费，是最终使用者查看结果的界面。

运行态需要承载以下能力：

- 图表卡片展示
- 表格视图展示
- 图层切换
- 图例开关
- 提示框交互
- 缩放与局部查看

运行态的重点是稳定、直观、低学习成本。

## ◆ 5. 模板体系设计

模板体系是整个产品方案的核心。平台中的所有图表都通过模板进入系统，而不是通过零散页面或单独功能入口接入。

### ● 模板的产品作用

模板需要同时回答以下问题：

- 这是什么图表
- 适用于什么场景
- 用户能配置哪些内容
- 默认展示效果是什么
- 最终以什么方式展示

### ● 本期模板清单

#### ① 折线图

用于时间序列趋势分析，是观察指标变化最基础的图表类型。适合展示单指标趋势、多指标趋势、双轴趋势以及统计指标叠加结果。

#### ② 柱状图

用于分类对比和结构比较，适合展示不同对象之间的数值差异，也适合展示堆叠关系和并列比较关系。

#### ③ 散点图

用于观察两个指标之间的关系，适合做分布分析、异常点识别和分类聚类展示。

#### ④ 表格

用于展示图表明细数据或补充结果。表格不仅是辅助视图，也可作为独立展示形态存在。

#### ⑤ 混合图

用于在同一个图表中混合展示折线、柱状、面积等不同形态，适合多个指标联动分析和不同量纲对比场景。

### ● 模板需要具备的产品属性

| 属性 | 作用 |
| --- | --- |
| 模板名称 | 供用户识别和选择 |
| 模板说明 | 说明模板适用范围 |
| 维度范围 | 限制用户可配置的维度数量 |
| 指标范围 | 限制用户可配置的指标数量 |
| 展示方式 | 规定支持折线、柱状、散点、表格等形态 |
| 图层支持 | 规定是否支持图层切换 |
| 统计支持 | 规定是否支持均值、标准差、分位点 |
| 默认配置 | 保证创建后可直接进入编辑和预览 |

## ◆ 6. 核心功能设计

### ● 图表配置能力

图表配置能力是设计态最核心的产品能力。用户需要在一个统一界面中完成图表的主要配置。

配置内容包括：

- 基础配置：标题、模板、数据模型
- 数据配置：维度、指标、系列拆分、过滤条件
- 图层配置：图层新增、命名、启停、绑定
- 样式配置：颜色、线宽、透明度、点位显示、平滑
- 交互配置：图例、提示框、缩放、滑动条
- 统计配置：均值线、1σ、2σ、分位点、滚动窗口年数

### ● 图层能力

图层能力用于支持同一图表下不同分析层的组织与切换。

图层能力主要体现在两个方面：

#### ① 设计态配置

用户可以新增图层、定义图层名称、设置启停状态，并将维度、指标和统计项绑定到一个或多个图层。

#### ② 运行态切换

用户在运行态切换图层后，系统只展示当前图层下可见的维度、指标和统计项。

### ● 统计指标能力

统计指标能力是本项目区别于普通展示型图表页面的重要能力之一。它不是简单地显示原始数据，而是在原始指标基础上增加辅助分析信息。

本期统计能力包括：

| 统计能力 | 说明 |
| --- | --- |
| 均值线 | 展示指标在滚动窗口内的平均水平 |
| 1σ 区间 | 展示指标在滚动窗口内的一倍标准差范围 |
| 2σ 区间 | 展示指标在滚动窗口内的两倍标准差范围 |
| 分位点线 | 展示当前值在滚动窗口样本中的百分位位置 |
| 滚动 N 年窗口 | 由用户输入 N 年，控制统计计算范围 |

## ◆ 7. 产品交互形态

### ● 设计态交互形态

建议采用“工具区 + 画布区 + 配置区”的三段式结构：

| 区域 | 作用 |
| --- | --- |
| 工具区 | 选择模板、新增图表 |
| 画布区 | 预览图表、调整布局 |
| 配置区 | 配置当前图表的全部参数 |

这样的结构有两个优点：

- 用户能始终看到图表结果
- 用户能围绕当前图表集中完成全部配置

### ● 运行态交互形态

运行态建议采用“卡片式看板”结构：

| 区域 | 作用 |
| --- | --- |
| 卡片标题区 | 展示图表名称和操作入口 |
| 图表展示区 | 展示图表或表格结果 |
| 图层切换区 | 切换不同图层结果 |
| 交互区 | 使用 tooltip、legend、缩放等能力 |

## ◆ 8. 本期版本范围

### ● 本期纳入范围

本期重点建设以下产品能力：

- 折线图、柱状图、散点图、表格、混合图
- 图表模板体系
- 设计态与运行态
- 图层配置与图层切换
- 统计指标配置与统计结果展示

### ● 本期暂不纳入范围

本期不纳入以下能力：

- 后端查询引擎
- 权限体系
- 审批与发布流程
- 智能推荐和高级分析模型

本期从产品角度应形成以下结果：

| 产出 | 说明 |
| --- | --- |

| 模板体系 | 图表按模板管理和扩展 |
| 设计态产品能力 | 用户可通过界面完成图表配置 |

## ◆ 10. 结论

本项目的产品方案核心不是单纯提供几种图表，而是建立一套图表生产与展示机制。  
通过模板体系、设计态、运行态、图层和统计指标能力，可以在本期形成一个具备基础产品能力的 BI 图表平台，并为后续扩展更丰富的图表和分析能力打下基础。
